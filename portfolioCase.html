Portfolio Redesign 2024

  1. #CSS
  2. #HTML
  3. #Bootstrap

Background

Autumn 2023 i enrolled in a new two-year frontend program and in preperation for my internship period in fall 2024 i wanted to revamp my old portfolio.

Mission

Create a stylish portfolio that reflects UX trends and use new technologies/techniques I have learned.

#note!
This isn't a case study and reads more like a blogpost. No real research went into this i just wanted to create something for myself.

Phase 1 Ideation - But why?

I built my previous portfolio during a two-and-a-half year UX program with vanilla html/css and while functional it wasn't very dynamic or interesting. Now enrolled in a frontend program , I have gotten much more confident building websites and so I wanted to give my portfolio a big refurbishing.

While i still find elements of my portfolio pretty cool i still thought it was largely outdated. I utilized neobrutalism, color-blocking and bold fonts/colors. My personal style has since changed and i want my portfolio to be a reflection of that.

Phase 2 - Taking action!

After recognizing that I wanted to redesign my portfolio I wasted no time in beginning my ardous journey...

Phase 3 - Facing the behemoth...

After formulating a layout and a idea of what i want to achieve it was not time to begin building my portfolio...

Since enrolling in my program i had gotten a much firmer grasp of html, css, javascript and Bootstrap so i sought to utilize these when redesigning my portfolio. I also considered what kind of layout I wanted to do and looked at tons of portfolios online for inspiration. I decided to stick to my old layout

I decided to use GitHub to host and manage my portfolio. I had very little experience with GitHub prior to enrolling in my program but i caught on fairly quick. I also discovered GitHub Pages and decided to host my portfolio there. It was a wise decision economically but it also made sense

I made the excecutive decision to use Bootstrap 5 for this project. It made the issue of responsiveness much easier to solve and made my CSS file a lot prettier.
Bootstrap came with its own issues though that i wasn't entirely equipped to solve on my own which made me rely on google a lot. One issue was the Bootstrap default styles. Eg. wanting to override a bs default background was a bit tricky for me but i solved it by adding a id tag to body so it made my selectors more specific.
